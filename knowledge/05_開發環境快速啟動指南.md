# HRMS 開發環境快速啟動指南 (Developer Quick Start Guide)

歡迎加入 HRMS (人力資源暨專案管理系統) 開發團隊！本指南將帶領您如何在個人電腦上快速建立起運行 14 個微服務所需的基礎設施，並將前端與後端服務跑起來進行除錯。

## 系統先決條件 (Prerequisites)

在開始之前，請確保您的開發機 (Windows/Mac/Linux) 已安裝以下工具：

1. **Docker Desktop** (包含 Docker Compose) - 用於啟動資料庫與中介軟體。
2. **Java JDK 21** - 建議使用 Eclipse Temurin 或 Amazon Corretto。
3. **Maven 3.9+** - Java 建置工具。
4. **Node.js 20+** 及 **npm/yarn** - 用於啟動 React 前端。
5. **IDE** - 推薦使用 IntelliJ IDEA 或 VS Code。

---

## 步驟一：啟動基礎設施 (Database & Middlewares)

本專案將複雜的 PostgreSQL、Redis 等依賴整合成了一個 `docker-compose.dev.yml`。

1. 開啟終端機 (Terminal)。
2. 切換到專案根目錄 (`hr-system-2/`)。
3. 執行以下指令一鍵啟動所有基礎設施：
   ```bash
   docker-compose -f docker-compose.dev.yml up -d
   ```
4. **驗證啟動狀態**：
   執行 `docker ps`，您應該會看到以下服務正在運行：
   * `hrms-postgres` (Port: 5432) - PostgreSQL 資料庫
   * `hrms-redis` (Port: 6379) - Redis 快取服務
   * `hrms-pgadmin` (Port: 5050) - (選用) 視覺化資料庫管理介面
   
   > 💡 **小提示**：您可以打開瀏覽器進入 `http://localhost:5050` (帳: admin@hrms.com / 密: admin123) 來直接用視覺化介面查看資料庫。

---

## 步驟二：啟動後端微服務 (Backend Services)

目前端只需要純 API 測試時，可以利用 Maven 一鍵啟動所需的微服務模組。

### 方法 A：使用終端機啟動單一服務 (如 IAM 模組)
```bash
cd backend
# 啟動 IAM 服務 (Port 8081)
mvn spring-boot:run -pl hrms-iam
```
*(注意：開發模式下預設採用 `dev` profile，會自動連線到您的本機 `localhost:5432` 的 PostgreSQL)*

### 方法 B：使用 IntelliJ IDEA (推薦)
1. 用 IDEA 開啟 `hr-system-2/backend` 資料夾。
2. 等待 Maven 自動載入依賴並解析 Parent POM。
3. 在上方選單設定 Run Configuration，選擇 `Spring Boot`。
4. Main Class 指定各模組的 Application (如：`IamApplication.java`, `OrganizationApplication.java`)。
5. 點擊 **Run** 或 **Debug** 即可啟動。

---

## 步驟三：啟動前端介面 (Frontend Vite)

前端採用 React + Vite 架構，並在內部配置了 Mock API 以利於脫離後端進行獨立開發測試。

1. 開啟新的終端機視窗，進入 frontend 目錄：
   ```bash
   cd frontend
   ```
2. 安裝必要的 npm 套件：
   ```bash
   npm install
   ```
3. 啟動 Vite 開發伺服器：
   ```bash
   npm run dev
   ```
4. 開啟瀏覽器進入 `http://localhost:5173`。

> 💡 ** मॉकAPI 機制 (Mock API Mode)**：
> 前端內建了 `MockConfig.ts` 機制。若後端微服務尚未完全啟動，或是您負責開發前端正在刻 UI，您可以直接依賴 Mock API 運作（假資料已全部備妥）。

## 步驟四：如何執行與除錯整合測試 (Running Tests)

本系統採用 TDD 流程，並有著嚴格的合約驅動測試 (Contract-Driven Testing)。在推送您的程式碼之前，**務必確保您修改的模組測試全數通過**。

以 `hrms-training` (培訓模組) 為例，執行整合測試：
```bash
cd backend
mvn test --projects hrms-training
```

**測試報錯除錯重點：**
1. 若拋出 `Constraint Violation`，請檢查 `xxx_test_data.sql` 中的字串是否拼錯了 Domain 中定義的 Enum。
2. 若拋出 `Type Mismatch (UUID vs String)`，請檢查 `QueryBuilder.where().eq()` 送進去的參數型別是否正確。
3. 測試環境預設使用記憶體資料庫 (H2)，測試結束即銷毀，不會污染您的本機 `hrms_postgres` 開發資料。

## 遇到問題？(Troubleshooting)

* **Q: PostgreSQL 啟動失敗，顯示 Port 5432 被占用？**
  A: 您的本機可能已經安裝了一套 PostgreSQL，請先將其停用，或修改 `docker-compose.dev.yml` 將對外的 Port 改為 `5433:5432`，並同步修改微服務內的 `application.yml`。
  
* **Q: 想開啟 Kafka 測試非同步功能？**
  A: 打開 `docker-compose.dev.yml`，將 `zookeeper` 與 `kafka` 區塊的註解 (comment line `#`) 移除並重新執行 `docker-compose up -d`。
