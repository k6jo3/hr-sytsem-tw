## 1. 工時管理服務 (07)

### 1.1 業務流程圖：工時回報與鎖定
```mermaid
flowchart TD
    Start([員工回報工時]) --> Select[選擇專案+工項]
    Select --> Input[輸入工作日期+時數]
    Input --> Validate{防呆檢查}
    
    Validate -->|未來日期| E1[錯誤：不可回報未來]
    Validate -->|超過24h| E2[錯誤：單日上限24h]
    Validate -->|非專案成員| E3[錯誤：無權限]
    Validate -->|通過| Submit[提交至PM審核]
    
    Submit --> PMReview{PM審核}
    PMReview -->|核准| Approve[更新Project工時]
    PMReview -->|駁回| Reject[通知員工]
    
    Approve --> MonthEnd{月底薪資結算?}
    MonthEnd -->|是| Lock[鎖定工時<br/>不可修改]
    MonthEnd -->|否| CanEdit[可繼續編輯]
```

### 1.2 循序圖：工時審核與專案成本更新
```mermaid
sequenceDiagram
    participant E as Employee
    participant T as Timesheet Service

    T->>T: 防呆檢查通過
    T->>W: 啟動審核流程
    W->>E: 通知PM審核
    
    Note over W: PM審核...
    
    W->>T: 審核通過回調
    T->>T: 標記approved
    T->>P: TimesheetApproved事件

    alt 月底薪資結算
        Pay->>T: PayrollRunCompleted事件
        T->>T: 鎖定當月工時
    end
```

```
UserDelegation {

delegator: 李經理
delegate: 王副理

endDate: 2025-12-20
delegationScope: ALL
```

#### 催辦提醒機制
```
定時Job每日檢查：
- 待辦超過3天 → 發送第一次提醒
- 待辦超過5天 → 發送第二次提醒+通知上級
- 待辦超過7天 → 升級至更高層主管
```

```

  "taskIds": ["uuid1", "uuid2", "uuid3"],
  "comments": "批次核准"
}

Response: {
  "approved": 3,
  "failed": 0
}
```

## 3. 通知服務 (12) - CROSS-001

### 3.1 完整架構設計

#### 通知偏好設定
```
UserNotificationPreference {
  userId: UUID
  
  channels: Map<NotificationType, List<Channel>>
  // 範例：
  // LEAVE_APPROVED → [EMAIL, IN_APP]
  // PAYSLIP_GENERATED → [EMAIL]
  // OVERTIME_ALERT → [EMAIL, PUSH, LINE]
  
  quietHours: QuietHours (勿擾時段)
  language: String (zh-TW, en-US)
}

QuietHours {
  enabled: Boolean
  startTime: Time (22:00)
  endTime: Time (08:00)
}
```

#### 企業通訊軟體整合

**Microsoft Teams整合:**
```java
@Service
public class TeamsNotificationService {
    
    public void sendToChannel(String webhookUrl, String message) {
        MessageCard card = MessageCard.builder()
            .title("系統通知")
            .text(message)
            .themeColor("0078D7")
            .build();
            
        restTemplate.postForEntity(webhookUrl, card, Void.class);
    }
}
```

**Slack整合:**

## 4. 資料匯入/匯出 (CROSS-002)

### 4.1 批次匯入設計

#### Excel範本格式
```
員工批次匯入範本.xlsx:

Column A: 員工編號* (必填)
Column B: 姓名* (必填)
Column C: 身分證號* (必填，加密儲存)
Column D: Email* (必填)
Column E: 手機
Column F: 部門代碼*
Column G: 職稱*
Column H: 月薪*
Column I: 到職日*
...

備註：
- 標*為必填欄位
- 身分證號格式：A123456789
- Email須唯一
- 部門代碼需先建立
```

#### 匯入API
```
POST /api/v1/employees/import
Content-Type: multipart/form-data

Request:
  file: employees.xlsx
  dryRun: true (預覽模式，不實際匯入)

Response:
{
  "totalRows": 50,
  "validRows": 45,
  "invalidRows": 5,
  "errors": [
    {
      "row": 3,
      "field": "email",
      "error": "Email重複"
    }
  ],
  "preview": [...]
}

若dryRun=false，則實際執行匯入
```

## 5. 文件管理服務 (13)

### 5.1 文件分類與權限

```
文件類型與可見性：

EMPLOYEE_CONTRACT (員工合約):
  - 擁有者：該員工+HR
  - 儲存位置：/storage/contracts/{employeeId}/
  
PAYSLIP (薪資單):
  - 擁有者：該員工+財務
  - 加密：AES-256
  - 密碼保護：員工身分證後4碼
  
EMPLOYEE_PHOTO (員工照片):
  - 擁有者：該員工+全公司（通訊錄）
  - 大小限制：2MB
  
POLICY_DOCUMENT (政策文件):
  - 擁有者：HR
  - 可見性：全公司
```

### 5.2 業務流程：產生在職證明
```mermaid
flowchart TD
    Start([員工申請在職證明]) --> Submit[提交申請表單]
    Submit --> HRReview{HR審核}
    HRReview -->|駁回| Reject[通知員工]
    HRReview -->|核准| Generate[調用Document Service]
    
    Generate --> Template[載入Word範本]
    Template --> Replace[替換變數]
    Replace --> PDF[轉換為PDF]
    PDF --> Store[儲存至Document Service]
    Store --> Notify[通知員工下載]
```
