### 3.3 看板狀態欄

| 欄位 | 狀態 | 說明 |
|:---|:---|:---|
| 履歷篩選 | `SCREENING` | 新收到的應徵者，待初步篩選 |
| 面試中 | `INTERVIEWING` | 已安排/進行中的面試 |
| Offer | `OFFER` | 通過面試，待發送/已發送 Offer |
| 已錄取 | `HIRED` | 接受 Offer，準備到職 |

### 3.4 操作事件規格

| 事件 ID | 觸發元素 | 行為 | 對應 API | UI 反應 |
|:---|:---|:---|:---|:---|
| E-REC-01 | 新增職缺按鈕 | 開啟職缺表單 Modal | `POST /api/v1/jobs` | 彈出 Modal |
| E-REC-02 | 卡片拖曳 | 更新應徵者狀態 | `PUT /api/v1/candidates/{id}/status` | 樂觀更新，失敗回復 |
| E-REC-03 | 查看按鈕 | 開啟應徵者詳情 | `GET /api/v1/candidates/{id}` | 彈出 Drawer |
| E-REC-04 | 安排面試 | 排程面試 | `POST /api/v1/interviews` | 彈出面試排程 Modal |
| E-REC-05 | 發送 Offer | 發送 Offer | `POST /api/v1/offers` | 彈出 Offer 表單 Modal |
| E-REC-06 | 建立按鈕 | 建立員工資料 | 發布 CandidateHiredEvent | 跨服務至 Organization |
| E-REC-07 | 職缺篩選 | 按職缺篩選看板 | — | 前端篩選 |

## 5. API 端點對照表

| 方法 | 端點 | 用途 | 對應畫面 |
|:---|:---|:---|:---|
| `GET` | `/api/v1/jobs` | 職缺列表 | P01 篩選, P02 |
| `POST` | `/api/v1/jobs` | 新增職缺 | P01 |
| `PUT` | `/api/v1/jobs/{id}` | 更新職缺 | P02 |
| `GET` | `/api/v1/candidates` | 應徵者列表 | P01 |
| `GET` | `/api/v1/candidates/{id}` | 應徵者詳情 | P01, P03 |
| `PUT` | `/api/v1/candidates/{id}/status` | 更新狀態 | P01 (拖曳) |
| `POST` | `/api/v1/interviews` | 安排面試 | P01, P04 |
| `PUT` | `/api/v1/interviews/{id}/evaluate` | 面試評估 | P04 |
| `POST` | `/api/v1/offers` | 發送 Offer | P01, P05 |
| `PUT` | `/api/v1/offers/{id}/accept` | 接受 Offer | P05 |
| `GET` | `/api/v1/recruitment/dashboard` | 招募儀表板 | P06 |
