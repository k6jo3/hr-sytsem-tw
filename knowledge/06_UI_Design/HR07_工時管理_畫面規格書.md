## 3. HR07-P01 每週工時報表

### 3.1 頁面概述

| 項目 | 說明 |
|:---|:---|
| **用途** | 員工按週填報工時至各專案/任務，提交審核 |
| **使用角色** | EMPLOYEE |
| **進入方式** | 側邊選單 → 工時申報 → 每週工時報表 |
| **頁面類型** | 表單頁（週選擇器 + 日曆表格） |

### 3.3 操作事件規格

| 事件 ID | 觸發元素 | 行為 | 對應 API | UI 反應 |
|:---|:---|:---|:---|:---|
| E-TSH-01 | 週選擇器 | 切換週次 | `GET /api/v1/timesheets/my?week={week}` | 表格更新 |
| E-TSH-02 | 新增工時項目 | 開啟 HR07-M01 | — | 彈出 Modal |
| E-TSH-03 | 儲存工時 | 儲存填報內容 | `POST /api/v1/timesheets/entries` | message.success |
| E-TSH-04 | 送審按鈕 | **`Modal.confirm` 確認對話框**（「確定要送出本週工時嗎？送出後將無法修改。」） → 提交本週工時 | `PUT /api/v1/timesheets/{id}/submit` | 確認後送審；狀態 → 待審核 |
| E-TSH-05 | 儲存格點擊 | 編輯工時值 | — | 原地可編輯 InputNumber |

### 3.4 防呆驗證

| 規則 | 說明 |
|:---|:---|
| 不可回報未來日期 | 未來日期儲存格禁用 |
| 單日工時 0~24h | 超過 24 小時提示錯誤 |
| 同日同專案不重複 | 同日期同任務只能一筆 |
| LOCKED 狀態不可編輯 | 已鎖定月份不可修改 |
