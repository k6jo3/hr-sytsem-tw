## 🔍 合約內容結構

每個場景的合約定義包含以下完整資訊：

### Command 操作合約

```json
{
  "scenarioId": "場景唯一識別碼",
  "apiEndpoint": "API 端點路徑",
  "controller": "對應的 Controller 類別",
  "service": "對應的 Service 實作類別",
  "permission": "所需權限",
  "request": { /* 請求範例 */ },
  "businessRules": [ /* 業務規則列表 */ ],
  "expectedDataChanges": [ /* 預期的資料異動 */ ],
  "expectedEvents": [ /* 預期的領域事件 */ ],
  "expectedResponse": { /* 預期的 API 回應（可選） */ }
}
```

**驗證三層：**
1. **業務規則：** 驗證輸入參數是否符合業務邏輯
2. **資料異動：** 驗證 INSERT/UPDATE/DELETE/SOFT_DELETE 操作
3. **領域事件：** 驗證發布的事件內容

### Query 操作合約

```json
{
  "scenarioId": "場景唯一識別碼",
  "apiEndpoint": "API 端點路徑",
  "controller": "對應的 Controller 類別",
  "service": "對應的 Service 實作類別",
  "permission": "所需權限",
  "request": { /* 查詢參數 */ },
  "expectedQueryFilters": [ /* 預期的過濾條件 */ ],
  "expectedResponse": { /* 預期的回應結構 */ }
}
```

**驗證兩層：**
1. **查詢過濾：** 驗證 QueryGroup 的過濾條件
2. **回應結果：** 驗證欄位、型別、格式、分頁、排序
