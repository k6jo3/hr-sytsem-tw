## ⚠️ 已知問題

### 1. 測試執行時 JVM 崩潰

**症狀：**
```
# There is insufficient memory for the Java Runtime Environment to continue.
```

**原因：** 記憶體不足

**解決方案：**
- 使用 `-DargLine="-Xmx512m"` 限制記憶體使用
- 或者分批執行測試

### 2. 測試文件中的斷言方法

目前測試使用的是 `assertContract()`，這是向下相容的方法，**只驗證查詢條件**。

**升級建議：**
修改測試以使用新的 `verifyQueryContract()`，可以驗證完整的輸入+輸出。
