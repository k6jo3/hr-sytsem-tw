**日期：** 2026-02-10

**狀態：** ✅ 完成

## 升級背景

原有的合約測試架構只能驗證 QueryGroup 的過濾條件，無法自動化驗證 API 回應結果和資料異動，導致合約測試不夠完整。

### 問題點
1. ❌ 預期輸出結果無法自動化驗證
2. ❌ 資料異動無法自動化驗證
3. ❌ 領域事件發布無法驗證
4. ❌ 測試重點只在「輸入」，不在「結果」

---

## 升級內容

### 1. 合約格式升級

**從 Markdown 表格升級為 JSON Schema 格式**

#### 舊格式（僅支援查詢條件）
```markdown
| 場景 ID | 測試描述 | 必須包含的過濾條件 |
| :--- | :--- | :--- |
| EMP_001 | 查詢在職人員 | `status = 'ACTIVE'`, `is_deleted = 0` |
```

#### 新格式（完全可驗證）
```json
{
  "scenarioId": "RPT_QRY_001",
  "apiEndpoint": "GET /api/v1/reports/hr/employee-roster",
  "expectedQueryFilters": [
    {"field": "employment_status", "operator": "=", "value": "ACTIVE"}
  ],
  "expectedResponse": {
    "statusCode": 200,
    "dataPath": "data.content",
    "minRecords": 1,
    "requiredFields": [
      {"name": "employeeId", "type": "uuid"},
      {"name": "nationalIdMasked", "type": "string", "format": "masked"},
      {"name": "serviceYears", "type": "decimal", "precision": 1}
    ],
    "orderBy": {"field": "employeeNumber", "direction": "ASC"},
    "pagination": {"required": true},
    "assertions": [
      {"field": "nationalIdMasked", "operator": "contains", "value": "*"}
    ]
  }
}
```

## 驗證能力

### 欄位型別驗證
- ✅ `uuid` - UUID 格式（36 字元，包含破折號）
- ✅ `string` - 字串型別
- ✅ `integer` - 整數型別
- ✅ `decimal` - 小數型別（可指定精度）
- ✅ `boolean` - 布林型別
- ✅ `date` - 日期格式（YYYY-MM-DD）
- ✅ `datetime` - 日期時間格式（ISO 8601）
- ✅ `email` - Email 格式
- ✅ `phone` - 電話號碼

### 特殊格式驗證
- ✅ `masked` - 遮罩格式（需包含 `*`）

### 斷言操作符
- ✅ `equals` / `notEquals` - 相等/不相等
- ✅ `contains` / `notContains` - 包含/不包含
- ✅ `greaterThan` / `lessThan` - 大於/小於
- ✅ `notNull` / `null` - 非空/空值

### 資料異動類型
- ✅ `INSERT` - 新增記錄
- ✅ `UPDATE` - 更新記錄
- ✅ `DELETE` - 刪除記錄（硬刪除）
- ✅ `SOFT_DELETE` - 軟刪除（設定 `is_deleted = true`）

## 使用範例

### Query 操作測試
```java
@Test
@DisplayName("RPT_QRY_001: 查詢在職人員名冊")
void testEmployeeRoster() throws Exception {
    // 1. 載入合約
    ContractSpec contract = loadContract("reporting", "RPT_QRY_001");

    // 2. 執行 API
    MvcResult result = mockMvc.perform(get("/api/v1/reports/hr/employee-roster")
            .param("status", "ACTIVE"))
            .andExpect(status().isOk())
            .andReturn();

    // 3. 捕獲 QueryGroup
    verify(repository).findPage(queryCaptor.capture(), any());

    // 4. 驗證完整合約
    verifyQueryContract(
        queryCaptor.getValue(),
        result.getResponse().getContentAsString(),
        contract
    );
}
```

### Command 操作測試
```java
@Test
@DisplayName("RPT_CMD_001: 建立個人儀表板")
void testCreateDashboard() throws Exception {
    // 1. 載入合約
    ContractSpec contract = loadContract("reporting", "RPT_CMD_001");

    // 2. 執行前快照
    Map<String, List<Map<String, Object>>> before = captureDataSnapshot("dashboards");

    // 3. 執行 API
    mockMvc.perform(post("/api/v1/reports/dashboards")
            .content(requestJson))
            .andExpect(status().isOk());

    // 4. 執行後快照
    Map<String, List<Map<String, Object>>> after = captureDataSnapshot("dashboards");

    // 5. 驗證完整合約
    verifyCommandContract(before, after, capturedEvents, contract);
}
```
