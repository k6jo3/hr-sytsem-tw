## ✅ 問題修正記錄

### 1. 合約檔案載入問題（已修正）

**問題：** 測試尋找 `iam_contracts_contracts.md` 而非 `iam_contracts.md`

**根因：** `BaseContractTest.loadContractSpec(serviceName)` 會自動加上 `_contracts.md` 後綴

**解決方案：**
- 修正所有測試檔案的合約載入參數
- `loadContractSpec("iam_contracts")` → `loadContractSpec("iam")`
- `loadContract("iam_contracts", scenarioId)` → `loadContract("iam", scenarioId)`

**修正檔案：**
- `IamApiContractTest.java`
- `AuthenticationApiContractTest.java`
- `PermissionApiContractTest.java`
- `ProfileApiContractTest.java`

### 2. Query 測試 API 升級（已完成）

**問題：** Query 測試使用舊版 Markdown 表格格式 API `assertContract()`

**解決方案：** 升級為新版 JSON Schema 格式 API

**修正模式：**
```java
// ❌ 舊版（Markdown 表格格式）
assertContract(queryCaptor.getValue(), contractSpec, "IAM_QRY_001");

// ✅ 新版（JSON Schema 格式）
ContractSpec contract = loadContract("iam", "IAM_QRY_001");
var result = mockMvc.perform(get("/api/v1/users?status=ACTIVE"))
        .andExpect(status().isOk())
        .andReturn();
String responseJson = result.getResponse().getContentAsString();
verifyQueryContract(queryCaptor.getValue(), responseJson, contract);
```

**升級數量：** 9 個 Query 測試全部升級完成

## 🔍 問題分類與分析

### 問題類型一：合約驗證失敗（13 個）⚠️

**這是好消息！** 合約測試正確發揮功能，抓到實作中缺少的安全性過濾條件。

#### 1.1 缺少軟刪除過濾（8 個場景）

**影響場景：**
- IAM_QRY_001, IAM_QRY_004, IAM_QRY_005 （User 查詢）
- IAM_QRY_101, IAM_QRY_102, IAM_QRY_103, IAM_QRY_104, IAM_QRY_105 （Role 查詢）

**合約要求：** 所有查詢必須加上 `is_deleted = false`

**實際產出：** 沒有 `is_deleted` 過濾條件

**範例錯誤訊息：**
```
╔══════════════════════════════════════════════════════════════╗
║                    合約驗證失敗                               ║
╠══════════════════════════════════════════════════════════════╣
║ 場景 ID: IAM_QRY_001                                          ║
╠══════════════════════════════════════════════════════════════╣
║ 缺失的過濾條件:                                               ║
║   ❌ is_deleted = false                                       ║
╠══════════════════════════════════════════════════════════════╣
║ 實際產出的過濾條件:                                           ║
║   ✓ status = 'ACTIVE'                                        ║
║   ✓ tenant_id = 'T001'                                       ║
╚══════════════════════════════════════════════════════════════╝
```

**修正建議：** 在 Service 層的 QueryBuilder 中加入軟刪除過濾

```java
// 範例：GetUserListServiceImpl
QueryGroup query = QueryBuilder.start()
    .eq("status", req.getStatus())
    .eq("tenant_id", currentUser.getTenantId())
    .eq("is_deleted", false)  // ← 加入此行
    .build();
```

#### 1.2 缺少多租戶過濾（6 個場景）

**影響場景：**
- IAM_QRY_001, IAM_QRY_004 （User 查詢）
- IAM_QRY_101, IAM_QRY_102, IAM_QRY_103, IAM_QRY_104 （Role 查詢）

**合約要求：** `tenant_id = {currentUserTenantId}`

**實際產出：** 有 `tenant_id = 'T001'`，但合約檢查時參數替換有問題

**分析：**
- 實作是正確的（有加上 tenant_id 過濾）
- 問題在於合約中使用 `{currentUserTenantId}` 佔位符
- 測試執行時沒有正確替換佔位符

**修正建議：** 在測試的 `setUp()` 中替換佔位符

```java
@BeforeEach
void setUp() throws Exception {
    mockAdminUser = new JWTModel();
    mockAdminUser.setTenantId("T001");

    contractSpec = loadContractSpec("iam");
    contractSpec = contractSpec.replace("{currentUserTenantId}", "T001");  // ← 替換佔位符
}
```

    role_id UUID REFERENCES roles(role_id),
