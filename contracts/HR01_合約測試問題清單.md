## 🔍 問題根因分析

### 根因一：安全性過濾條件缺失（8 個場景）🔴

**影響範圍：** IAM_QRY_001, 004, 005, 101, 102, 103, 104, 105

**問題描述：**
所有 Query Service 都缺少 `is_deleted = false` 軟刪除過濾條件。

**風險評估：**
- 🔴 **高風險** - 可能查詢到已刪除的資料
- 🔴 **資料洩漏** - 違反業務邏輯

**範例錯誤：**
```
╔══════════════════════════════════════════════════════════════╗
║ 場景 ID: IAM_QRY_001                                          ║
╠══════════════════════════════════════════════════════════════╣
║ 缺失的過濾條件:                                               ║
║   ❌ is_deleted = false                                       ║
║   ❌ tenant_id = {currentUserTenantId}                        ║
╠══════════════════════════════════════════════════════════════╣
║ 實際產出的過濾條件:                                           ║
║   ✓ status = 'ACTIVE'                                        ║
║   ✓ tenant_id = 'T001'                                       ║
╚══════════════════════════════════════════════════════════════╝
```

**修正方案：**

需修正以下 8 個 Service 檔案：

1. `GetUserListServiceImpl.java` (IAM_QRY_001)
2. `GetUserServiceImpl.java` (IAM_QRY_002) - 如果存在
3. `GetLockedUsersServiceImpl.java` (IAM_QRY_004) - 或在 GetUserListServiceImpl 中
4. `GetUsersByTenantServiceImpl.java` (IAM_QRY_005)
5. `GetRoleListServiceImpl.java` (IAM_QRY_101, 102, 103, 104)
6. `GetRoleServiceImpl.java` (IAM_QRY_105)

**修正代碼範例：**

```java
// ❌ 錯誤 - 缺少 is_deleted 過濾
@Override
public UserListResponse getResponse(GetUserListRequest req, JWTModel currentUser) {
    QueryGroup query = QueryBuilder.start()
        .eq("status", req.getStatus())
        .eq("tenant_id", currentUser.getTenantId())
        .build();

    Page<User> users = userRepository.findPage(query, req.toPageable());
    return UserListResponse.from(users);
}

// ✅ 正確 - 加入 is_deleted 過濾
@Override
public UserListResponse getResponse(GetUserListRequest req, JWTModel currentUser) {
    QueryGroup query = QueryBuilder.start()
        .eq("status", req.getStatus())
        .eq("tenant_id", currentUser.getTenantId())
        .eq("is_deleted", false)  // ← 加入此行
        .build();

    Page<User> users = userRepository.findPage(query, req.toPageable());
    return UserListResponse.from(users);
}
```

### 根因二：API 端點缺失（1 個）🔴

**影響場景：** IAM_QRY_002

**問題描述：**
`GET /api/v1/users/{id}` 端點不存在，回傳 HTTP 404。

**風險評估：**
- 🔴 **功能缺失** - 無法查詢單一使用者詳情

**修正方案：**

在 `HR01UserQryController.java` 新增端點：

```java
@GetMapping("/{id}")
@Operation(summary = "查詢使用者詳情", operationId = "getUser")
public ResponseEntity<UserDetailResponse> getUser(
        @PathVariable String id,
        @CurrentUser JWTModel currentUser) throws Exception {
    GetUserRequest request = new GetUserRequest(id);
    return ResponseEntity.ok(getResponse(request, currentUser));
}
```

實作 `GetUserServiceImpl.java`：

```java
@Service("getUserServiceImpl")
public class GetUserServiceImpl
        implements QueryApiService<GetUserRequest, UserDetailResponse> {

    @Autowired
    private IUserRepository userRepository;

    @Override
    public UserDetailResponse getResponse(GetUserRequest req, JWTModel currentUser) {
        QueryGroup query = QueryBuilder.start()
            .eq("user_id", req.getUserId())
            .eq("is_deleted", false)  // ← 軟刪除過濾
            .build();

        User user = userRepository.findAll(query).stream()
            .findFirst()
            .orElseThrow(() -> new UserNotFoundException(req.getUserId()));

        return UserDetailResponse.from(user);
    }
}
```

### 根因三：權限配置錯誤（4 個）🟡

**影響場景：** IAM_CMD_002, 003, 004, 006

**可能原因：**
1. Controller 有 `@PreAuthorize` 註解且測試 Mock 不正確

3. Test Security Context 設定不完整

```java
// 檢查這些 Controller 方法

```
