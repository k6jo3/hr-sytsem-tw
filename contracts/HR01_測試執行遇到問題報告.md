# HR01 IAM 服務 - 測試執行遇到問題報告

**日期：** 2026-02-11
**狀態：** ⚠️ 測試執行遇到記憶體不足問題
**階段：** 第三階段驗證 - 執行合約測試

---

## ⚠️ 問題描述

### 錯誤訊息

```
OpenJDK 64-Bit Server VM warning: INFO: os::commit_memory(0x0000000710200000, 58720256, 0) failed;
error='分頁檔太小，無法完成此操作。' (DOS error/errno=1455)
#
# There is insufficient memory for the Java Runtime Environment to continue.
# Native memory allocation (mmap) failed to map 58720256 bytes. Error detail: G1 virtual space
```

### 問題分析

1. **JVM 記憶體不足** - 嘗試分配 58MB 記憶體失敗
2. **分頁檔不足** - Windows 虛擬記憶體（分頁檔）太小
3. **測試編譯階段** - 在 `mvn test-compile` 時發生，尚未執行測試

### 嘗試過的解決方案

| 嘗試 | 命令 | 結果 |
|:---|:---|:---:|
| 1 | `mvn test -Dtest=*ApiContractTest` | ❌ 記憶體不足 |
| 2 | `set MAVEN_OPTS=-Xmx512m && mvn test-compile` | ❌ 記憶體不足 |
| 3 | `-DargLine="-Xmx256m"` | ❌ 記憶體不足 |

---

## 💡 解決方案建議

### 方案一：增加 Windows 虛擬記憶體（推薦）⭐

**步驟：**

1. **開啟系統內容**
   - 按 `Win + Pause/Break` 或右鍵「本機」→「內容」

2. **進階系統設定**
   - 點選「進階系統設定」
   - 切換到「進階」標籤
   - 點選「效能」區塊的「設定」按鈕

3. **調整虛擬記憶體**
   - 切換到「進階」標籤
   - 點選「虛擬記憶體」區塊的「變更」按鈕
   - 取消勾選「自動管理所有磁碟機的分頁檔大小」
   - 選擇系統磁碟（通常是 C:）
   - 選擇「自訂大小」
   - **初始大小：** 4096 MB (4GB)
   - **最大值：** 8192 MB (8GB)
   - 點選「設定」然後「確定」
   - **重新開機**使設定生效

### 方案二：分批執行測試

**逐一執行測試類別，避免一次載入太多：**

```bash
cd D:\git\hr-system2\hr-sytsem-2\backend\hrms-iam

# 1. 先編譯主程式碼
mvn clean compile

# 2. 分別編譯和執行各測試類別
mvn test -Dtest=IamApiContractTest
mvn test -Dtest=AuthenticationApiContractTest
mvn test -Dtest=PermissionApiContractTest
mvn test -Dtest=ProfileApiContractTest
```

### 方案三：使用 Gradle（如果可用）

```bash
cd D:\git\hr-system2\hr-sytsem-2\backend\hrms-iam
gradle test --tests "*ApiContractTest"
```

### 方案四：在 IDE 中執行

**使用 IntelliJ IDEA 或 Eclipse：**

1. 開啟專案
2. 右鍵點選測試類別
3. 選擇「Run 'IamApiContractTest'」
4. IDE 通常有更好的記憶體管理

### 方案五：增加 Maven 記憶體配置

**建立或修改 `.mvn/jvm.config` 檔案：**

```bash
# 在專案根目錄建立 .mvn/jvm.config
cd D:\git\hr-system2\hr-sytsem-2\backend\hrms-iam
mkdir .mvn
echo -Xms256m -Xmx512m -XX:MaxMetaspaceSize=256m > .mvn/jvm.config
```

然後重新執行測試。

## 🔧 其他已知問題

### 1. 測試資料庫未設定

**問題：** 合約測試需要連接資料庫來驗證資料異動和查詢條件。

**影響：** Command 測試的 `captureDataSnapshot()` 會失敗。

**解決方案：**
- 使用 Testcontainers 建立 PostgreSQL 測試容器
- 或使用 H2 In-Memory 資料庫
- 或連接本地 PostgreSQL 測試資料庫

**配置範例（application-test.yml）：**
```yaml
spring:
  datasource:
    url: jdbc:postgresql://localhost:5432/hrms_iam_test
    username: test
    password: test
  jpa:
    hibernate:
      ddl-auto: create-drop
```

### 2. 事件監聽器未實作

**問題：** Command 測試中的 `capturedEvents` 是空的（TODO 標記）。

**影響：** 無法驗證領域事件是否正確發布。

**解決方案：** 建立測試專用事件監聽器

```java
@Component
@TestComponent
public class TestEventListener {
    private List<Map<String, Object>> capturedEvents = new ArrayList<>();

    @EventListener
    public void onUserCreated(UserCreatedEvent event) {
        Map<String, Object> eventData = new HashMap<>();
        eventData.put("eventType", "UserCreatedEvent");
        eventData.put("userId", event.getUserId());
        eventData.put("username", event.getUsername());
        capturedEvents.add(eventData);
    }

    // ... 其他事件監聽器

    public List<Map<String, Object>> getCapturedEvents() {
        return new ArrayList<>(capturedEvents);
    }

    public void clear() {
        capturedEvents.clear();
    }
}
```

### 3. Repository 實作可能不完整

**問題：** 雖然 Service 層已實作，但 Infrastructure 層（Repository 實作）可能未完整。

**需要檢查：**
- UserRepositoryImpl
- RoleRepositoryImpl
- PermissionRepositoryImpl
- MyBatis Mapper XML 或 JPA Entity
