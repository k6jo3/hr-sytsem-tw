# 簽核流程服務(Workflow Service) 需求分析書

**版本:** 1.0  
**所屬領域:** 通用領域 (Generic Domain)  
**導入階段:** 第一階段（核心基礎服務）

---

## 1. 服務概述

### 1.1 核心職責
- 可視化流程設計器（拖拉式）
- 流程引擎執行
- 多級簽核、條件分流、平行會簽
- 代理人機制
- 簽核逾時提醒

---

## 2. 領域模型

### 2.1 聚合根

#### WorkflowDefinition (流程定義)
```
WorkflowDefinition {
  definitionId: UUID
  flowName: String
  flowType: FlowType (LEAVE_APPROVAL, OVERTIME_APPROVAL, PURCHASE...)
  
  nodes: List<WorkflowNode> (流程節點)
  edges: List<WorkflowEdge> (流程連線)
  
  isActive: Boolean
  version: Integer
}

WorkflowNode {
  nodeId: String
  nodeType: NodeType (START, APPROVAL, CONDITION, END)
  name: String
  config: JSON (審核人設定、條件設定等)
}

enum NodeType {
  START        // 開始
  APPROVAL     // 審核節點
  CONDITION    // 條件分流
  PARALLEL     // 平行會簽
  END          // 結束
}
```

#### WorkflowInstance (流程實例)
```
WorkflowInstance {
  instanceId: UUID
  definitionId: UUID (FK)
  businessType: String (leave, overtime...)
  businessId: UUID (對應業務單據ID)
  
  applicant: UUID (FK to Employee)
  currentNode: String
  
  status: InstanceStatus (RUNNING, COMPLETED, REJECTED, CANCELLED)
  
  startedAt: DateTime
  completedAt: DateTime
}

enum InstanceStatus {
  RUNNING
  COMPLETED
  REJECTED
  CANCELLED
}
```

#### ApprovalTask (審核任務)
```
ApprovalTask {
  taskId: UUID
  instanceId: UUID (FK)
  nodeId: String
  
  assignee: UUID (FK to Employee, 審核人)
  delegatedTo: UUID (nullable, 代理人)
  
  status: TaskStatus (PENDING, APPROVED, REJECTED)
  
  approvedAt: DateTime (nullable)
  comments: String
  
  dueDate: DateTime (nullable)
  isOverdue: Boolean
}

enum TaskStatus {
  PENDING
  APPROVED
  REJECTED
}
```

## 3. 核心API

### 3.1 啟動流程
```
POST /api/v1/workflows/start
Request:
{
  "flowType": "LEAVE_APPROVAL",
  "businessId": "uuid",
  "applicant": "uuid",
  "variables": {
    "leaveDays": 2,
    "leaveType": "ANNUAL"
  }
}

Response 201:
{
  "instanceId": "uuid",
  "currentNode": "manager_approval",
  "assignee": "uuid (直屬主管)"
}
```

### 3.2 審核任務
```
PUT /api/v1/workflows/tasks/{taskId}/approve
Request:
{
  "approverId": "uuid",
  "comments": "同意"
}

Response 200:
{
  "taskId": "uuid",
  "status": "APPROVED",
  "nextNode": "hr_approval" (或 "END" 表示結束)
}
```

**業務邏輯:**
- 更新ApprovalTask狀態
- 推進WorkflowInstance到下一節點
- 若結束，發布 `ApprovalCompleted` 事件
- 若駁回，發布 `ApprovalRejected` 事件

---

## 4. 領域事件

| 事件 | 觸發時機 | 訂閱服務 |
|:---|:---|:---|
| `ApprovalCompleted` | 流程核准完成 | Attendance, Recruitment等業務服務 |
| `ApprovalRejected` | 流程駁回 | Notification |
| `TaskAssigned` | 任務指派 | Notification |

## 5. 技術選型考量

**選項1: 自建流程引擎**
- 優點: 完全客製化、輕量
- 缺點: 開發成本高

**選項2: 整合Activiti/Camunda**
- 優點: 成熟的BPMN引擎、可視化設計器
- 缺點: 學習曲線、較重

**建議:** 第一期簡化實作（固定流程），第二期評估Activiti
