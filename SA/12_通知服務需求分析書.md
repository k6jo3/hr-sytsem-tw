# 通知服務(Notification Service) 需求分析書

**版本:** 1.0  
**所屬領域:** 通用領域 (Generic Domain)  
**導入階段:** 第一階段（核心基礎服務）

---

## 1. 服務概述

### 1.1 核心職責
- 多渠道通知（Email、系統訊息、推播、Teams/Slack/LINE）
- 通知範本管理
- 自動提醒事件（生日、合約到期、證照到期等）  
- 通知偏好設定
- 訊息佇列處理（確保送達）

## 2. 領域模型

### 2.1 聚合根

#### Notification (通知)
```
Notification {
  notificationId: UUID
  recipientId: UUID (FK to Employee)
  
  title: String
  content: Text
  notificationType: NotificationType
  
  channels: List<NotificationChannel>
  
  priority: Priority (LOW, NORMAL, HIGH, URGENT)
  
  status: NotificationStatus (PENDING, SENT, FAILED, READ)
  
  sentAt: DateTime (nullable)
  readAt: DateTime (nullable)
  
  relatedBusinessType: String (nullable, leave, overtime...)
  relatedBusinessId: UUID (nullable)
  
  createdAt: DateTime
}

enum NotificationType {
  APPROVAL_REQUEST     // 審核請求
  APPROVAL_RESULT      // 審核結果
  REMINDER            // 提醒
  ANNOUNCEMENT        // 公告
  ALERT               // 警示
}

enum NotificationChannel {
  IN_APP     // 系統內訊息
  EMAIL      // Email
  PUSH       // 推播
  TEAMS      // Microsoft Teams
  SLACK      // Slack
  LINE       // LINE
}

enum Priority {
  LOW
  NORMAL
  HIGH
  URGENT
}

enum NotificationStatus {
  PENDING
  SENT
  FAILED
  READ
}
```

#### NotificationTemplate (通知範本)
```
NotificationTemplate {
  templateId: UUID
  templateCode: String (unique)
  templateName: String
  
  subject: String (支援變數 {{name}})
  body: Text (支援變數)
  
  defaultChannels: List<NotificationChannel>
  
  isActive: Boolean
  createdAt: DateTime
}
```

## 3. Event-Driven架構

### 3.1 訂閱業務事件（關鍵！）

Notification Service訂閱**所有**業務事件，根據事件類型發送通知：

| 業務事件 | 通知對象 | 通知內容 |
|:---|:---|:---|
| `LeaveApplied` | 直屬主管 | "員工{{name}}申請{{days}}天請假，請審核" |
| `LeaveApproved` | 申請人 | "您的請假申請已核准" |
| `LeaveRejected` | 申請人 | "您的請假申請被駁回，原因:{{reason}}" |
| `OvertimeApplied` | 直屬主管 | "員工{{name}}申請{{hours}}小時加班" |
| `UserLocked` | 被鎖定使用者 | "您的帳號已被鎖定，請聯絡系統管理員" |
| `PayslipGenerated` | 員工 | "您的{{month}}月薪資單已產生" |
| `AnnualLeaveExpiring` | 員工+主管 | "員工{{name}}有{{days}}天特休將於{{date}}到期" |
| `ContractExpiring` | HR | "員工{{name}}的合約將於{{date}}到期" |
| `CertificateExpiring` | 員工 | "您的{{cert}}證照將於{{date}}到期" |

## 4. 核心API

### 4.1 發送通知API（內部服務調用）

```
POST /api/v1/notifications/send
Request:
{
  "recipientIds": ["uuid1", "uuid2"],
  "templateCode": "LEAVE_APPROVED",
  "variables": {
    "employeeName": "張三",
    "leaveDays": 2,
    "leaveType": "特休"
  },
  "channels": ["IN_APP", "EMAIL"],
  "priority": "NORMAL"
}

Response 202 Accepted:
{
  "notificationIds": ["uuid1", "uuid2"],
  "message": "通知已加入傳送佇列"
}
```

**業務邏輯（異步）:**
1. 根據templateCode載入範本
2. 替換變數生成最終內容
3. 加入訊息佇列
4. 依序發送各渠道
5. 更新發送狀態

### 4.2 查詢個人通知

```
GET /api/v1/notifications/me?status=UNREAD
Response 200:
[
  {
    "notificationId": "uuid",
    "title": "請假審核",
    "content": "員工張三申請2天請假，請審核",
    "type": "APPROVAL_REQUEST",
    "priority": "NORMAL",
    "status": "SENT",
    "sentAt": "2025-11-24T10:00:00Z",
    "relatedBusinessType": "leave",
    "relatedBusinessId": "uuid"
  }
]
```

### 4.3 標記為已讀

```
PUT /api/v1/notifications/{id}/read
Response 200:
{
  "notificationId": "uuid",
  "status": "READ",
  "readAt": "2025-11-24T11:00:00Z"
}
```

## 5. 多渠道實作

### 5.1 Email通知
- 技術: Spring Mail + SMTP
- 支援HTML格式
- 附件支援

### 5.2 系統內訊息
- 儲存於資料庫
- WebSocket推送即時通知
- 前端顯示小鈴鐺提示

### 5.3 推播通知
- 技術: Firebase Cloud Messaging (FCM)
- iOS/Android App支援

### 5.4 企業通訊軟體（選配）
- Teams: Teams Webhook
- Slack: Slack Bot API
- LINE: LINE Notify API
