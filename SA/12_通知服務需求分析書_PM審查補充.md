## 1. 通知偏好設定

### 1.1 UserNotificationPreference聚合根
```
UserNotificationPreference {
  userId: UUID
  
  // 通知渠道偏好（按事件類型）
  channelPreferences: Map<NotificationType, List<Channel>>
  
  // 勿擾時段
  quietHours: QuietHours
  
  // 語言偏好
  language: String (zh-TW, en-US)
  
  // 全域開關
  isEnabled: Boolean
  
  updatedAt: DateTime
}

QuietHours {
  enabled: Boolean
  startTime: Time (預設: 22:00)
  endTime: Time (預設: 08:00)
  daysOfWeek: List<DayOfWeek> (可指定特定日期)
}

enum NotificationType {
  LEAVE_APPLIED
  LEAVE_APPROVED
  OVERTIME_ALERT
  PAYSLIP_GENERATED
  PASSWORD_EXPIRING_SOON
  PROJECT_BUDGET_ALERT
  // ... 更多通知類型
}

enum Channel {
  EMAIL
  IN_APP      // 系統內通知
  PUSH        // 手機推播
  TEAMS       // Microsoft Teams
  SLACK
  LINE
}
```

#### 發送實作
```java
@Service
public class TeamsNotificationService implements NotificationChannel {
    
    public void send(NotificationMessage message, String webhookUrl) {
        TeamsMessageCard card = TeamsMessageCard.builder()
            .title(message.getTitle())
            .summary(message.getSummary())
            .themeColor(getThemeColor(message.getPriority()))
            .sections(buildSections(message))
            .potentialAction(buildActions(message))
            .build();
        
        HttpHeaders header = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        HttpEntity<TeamsMessageCard> request = new HttpEntity<>(card, headers);
        
        restTemplate.postForEntity(webhookUrl, request, Void.class);
    }
    
    private String getThemeColor(Priority priority) {
        return switch (priority) {
            case URGENT -> "FF0000"; // 紅色
            case HIGH -> "FFA500";   // 橘色
            case NORMAL -> "0078D7";  // 藍色
            case LOW -> "808080";    // 灰色
        };
    }
}
```

### 2.2 LINE Notify整合

#### Access Token取得
```
1. 訪問 https://notify-bot.line.me/
2. 登入LINE帳號
3. 發行存取權杖(Access Token)
4. 選擇通知的聊天室/群組
5. 將Access Token儲存至系統配置
```

#### 發送實作
```java
public void sendLineNotification(String message, String accessToken) {
    String url = "https://notify-api.line.me/api/notify";
    
    HttpHeaders headers = new HttpHeaders();
    headers.setBearerAuth(accessToken);
    headers.setContentType(MediaType.APPLICATION_FORM_URLENCODED);
    
    MultiValueMap<String, String> body = new LinkedMultiValueMap<>();
    body.add("message", message);
    
    HttpEntity<MultiValueMap<String, String>> request = new HttpEntity<>(body, headers);
    
    restTemplate.postForEntity(url, request, Void.class);
}
```

#### 訊息格式

```
期間：2025-12-10 ~ 2025-12-12
天數：3天
核准人：李經理
剩餘假期：5天
```

### 3.2 事件處理邏輯
```java
@Service
@Slf4j
public class NotificationEventHandler {
    
    @KafkaListener(topics = "${kafka.topics.domain-events}")
    public void handleDomainEvent(DomainEvent event) {
        try {
            // 1. 識別事件類型
            String eventType = event.getEventType();
            
            // 2. 查找通知配置
            NotificationConfig config = configRepo.findByEventType(eventType);
            if (config == null) {
                return; // 無需通知的事件
            }
            
            // 3. 確定通知對象
            List<UUID> recipients = determineRecipients(event, config);
            
            // 4. 為每個接收者發送通知
            for (UUID recipientId : recipients) {
                // 查詢使用者偏好
                UserNotificationPreference pref = prefRepo.findByUserId(recipientId);
                
                // 檢查勿擾時段
                if (isQuietHours(pref)) {
                    scheduleForLater(event, recipientId); // 稍後發送
                    continue;
                }
                
                // 取得通知範本
                NotificationTemplate template = templateRepo.findById(config.getTemplateId());
                
                // 渲染訊息內容
                String content = templateEngine.render(template, event.getPayload());
                
                // 根據偏好選擇渠道發送
                List<Channel> channels = pref.getChannelPreferences().get(eventType);
                for (Channel channel : channels) {
                    sendViaChannel(channel, recipientId, content);
                }
            }
            
        } catch (Exception e) {
            log.error("處理通知事件失敗: {}", event, e);
            // 重試機制...
        }
    }
}
```

## 4. 通知範本管理

### 4.1 範本結構
```
NotificationTemplate {
  templateId: String (TMPL_LEAVE_APPROVED)
  templateName: String
  category: TemplateCategory
  
  subject: String (可包含變數)
  body: Text (可包含變數)
  
  variables: List<String> (可用變數清單)
  
  supportedChannels: List<Channel>
  
  isActive: Boolean
}
```

### 4.2 範本範例：請假核准通知
```
templateId: TMPL_LEAVE_APPROVED
subject: 請假申請核准通知 - {{leaveTypeName}}
body: |
  親愛的 {{employeeName}}，

  您申請的{{leaveTypeName}}已獲核准：
  
  假別：{{leaveTypeName}}
  期間：{{startDate}} ~ {{endDate}}
  天數：{{totalDays}}天
  核准人：{{approverName}}
  核准時間：{{approvedAt}}
  
  剩餘假期餘額：{{remainingDays}}天
  
  ---
  此為系統自動通知，請勿直接回覆。
  如有疑問請聯繫您的主管或HR部門。

variables: [
  employeeName, leaveTypeName, startDate, endDate, 
  totalDays, approverName, approvedAt, remainingDays
]

supportedChannels: [EMAIL, IN_APP, PUSH]
```

### 4.3 變數替換引擎
```java
public String renderTemplate(NotificationTemplate template, Map<String, Object> data) {
    String content = template.getBody();
    
    for (Map.Entry<String, Object> entry : data.entrySet()) {
        String placeholder = "{{" + entry.getKey() + "}}";
        String value = entry.getValue().toString();
        content = content.replace(placeholder, value);
    }
    
    return content;
}
```
