# 工時管理服務(Timesheet Service) 需求分析書

**版本:** 1.0  
**日期:** 2025-11-24  
**所屬領域:** 核心領域 (Core Domain) - 專案成本管理  
**導入階段:** 第二階段（專案管理核心）

---

## 1. 服務概述與職責

### 1.1 核心職責
- 工時回報（日/週回報）
- 工時審核流程（PM審核）
- 工時防呆機制
- 工時統計分析（個人、專案、部門）
- 工時鎖定（已結算不可修改）  
- **專案成本核算的數據來源**

### 1.2 服務邊界
**屬於:** 工時回報、工時審核、工時統計  
**不屬於:** 專案定義(Project Service)、成本計算(需整合薪資)

## 2. 領域模型

### 2.1 聚合根

#### Timesheet (工時表)
```
Timesheet {
  timesheetId: UUID (PK)
  employeeId: UUID (FK)
  
  timesheetPeriod: TimesheetPeriod (DAILY, WEEKLY)
  periodStartDate: Date
  periodEndDate: Date
  
  entries: List<TimesheetEntry>
  
  totalHours: Decimal (自動計算)
  
  status: TimesheetStatus (DRAFT, SUBMITTED, APPROVED, REJECTED, LOCKED)
  
  submittedAt: DateTime (nullable)
  approvedBy: UUID (nullable, FK to Employee)
  approvedAt: DateTime (nullable)
  rejectionReason: String (nullable)
  
  isLocked: Boolean (已結算鎖定)
  
  createdAt: DateTime
  updatedAt: DateTime
}

enum TimesheetPeriod {
  DAILY   // 每日回報
  WEEKLY  // 每週回報
}

enum TimesheetStatus {
  DRAFT      // 草稿
  SUBMITTED  // 已提交
  APPROVED   // 已核准
  REJECTED   // 已駁回
  LOCKED     // 已鎖定
}
```

#### TimesheetEntry (工時明細)
```
TimesheetEntry {
  entryId: UUID (PK)
  timesheetId: UUID (FK)
  
  projectId: UUID (FK)
  taskId: UUID (nullable, FK, 可選擇具體工項)
  
  workDate: Date
  hours: Decimal
  description: String (工作說明)
  
  createdAt: DateTime
}
```

## 3. 核心業務邏輯

### 3.1 工時防呆規則
```
1. 不可回報未來日期的工時
2. 每日工時總和不可超過24小時
3. 同一天同一專案不可重複回報
4. 已鎖定的工時不可修改
5. 非專案成員不可回報該專案工時
```

### 3.2 工時審核流程
```
員工提交 → PM審核 → 核准/駁回
核准後 → 發布 TimesheetApproved event
→ Project Service更新Task工時
→ Payroll Service計算專案成本
```

### 3.3 工時鎖定機制
```
每月薪資結算完成後：
1. Payroll Service發布 PayrollRunCompleted event
2. Timesheet Service鎖定該月所有已核准工時
3. 鎖定後不可修改（避免影響已結算成本）
```

## 4. 領域事件

| 事件 | 觸發時機 | 訂閱服務 |
|:---|:---|:---|
| `TimesheetSubmitted` | 提交工時 | Workflow, Notification |
| `TimesheetApproved` | 審核通過 | Project, Payroll |
| `TimesheetRejected` | 審核駁回 | Notification |
| `TimesheetLocked` | 工時鎖定 | - |

## 5. 核心API

### 5.1 工時回報API

#### 提交工時（員工）
```
POST /api/v1/timesheets
Request:
{
  "employeeId": "uuid",
  "periodStartDate": "2025-11-18",
  "periodEndDate": "2025-11-24",
  "entries": [
    {
      "projectId": "uuid",
      "taskId": "uuid",
      "workDate": "2025-11-18",
      "hours": 8,
      "description": "開發登入功能"
    },
    {
      "projectId": "uuid2",
      "workDate": "2025-11-19",
      "hours": 6,
      "description": "需求討論會議"
    }
  ]
}

Response 201:
{
  "timesheetId": "uuid",
  "totalHours": 14,
  "status": "SUBMITTED"
}
```

#### 審核工時（PM）
```
PUT /api/v1/timesheets/{id}/approve
Request:
{
  "approverId": "uuid"
}

Response 200:
{
  "timesheetId": "uuid",
  "status": "APPROVED",
  "approvedAt": "2025-11-25T10:00:00Z"
}
```

**後續動作:**
- 發布 `TimesheetApproved` 事件
- Project Service更新Task.actualHours
- Payroll Service記錄專案成本數據

### 5.3 工時提醒API

#### 未回報提醒（定期Job執行）
```
GET /api/v1/timesheets/unreported?date={date}
Response 200:
[
  {
    "employeeId": "uuid",
    "employeeName": "李四",
    "unreportedDays": 3
  }
]
```

**業務邏輯:**
- 每日檢查前一天工時是否回報
- 發布事件給Notification Service發送提醒

## 6. 與其他服務整合

### 6.1 同步整合
- Project Service → 查詢專案與工項清單
- Payroll Service → 提供工時數據計算專案成本

### 6.2 異步整合
- 發布 `TimesheetApproved` → Project, Payroll
- 訂閱 `PayrollRunCompleted` (Payroll) → 鎖定工時
- 訂閱 `ProjectMemberRemoved` (Project) → 停止回報權限

## 7. 非功能需求

| 需求 | 目標 |
|:---|:---|
| 工時回報率 | >95% |
| 工時回報即時性 | 當日或次日 |
| 工時數據準確率 | >99% |
| 查詢回應時間 | <200ms |
