## 0. 前言與架構決策

### 重要決策
1. **Repository 不做整合測試** - 根據架構決策，Repository 層已由 MyBatis/Querydsl 保證正確性，不需額外整合測試
2. **合約測試的重點** - 合約測試重點在於案例的正確性，而不是 SQL 的生成是否完全符合
3. **測試優先級** - API 整合測試 > 合約測試 > Unit Test
4. **測試結果處理** - 不通過的測試，留下TODO:測試失敗
