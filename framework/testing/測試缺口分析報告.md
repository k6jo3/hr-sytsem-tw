## 5. 測試資料腳本範本

每個模組應建立以下測試資料腳本：

```
src/test/resources/
├── test-data/
│   ├── {module}_base_data.sql       # 基礎資料 (FK 依賴)
│   ├── {module}_query_test.sql      # 查詢測試資料
│   ├── {module}_crud_test.sql       # CRUD 測試資料
│   └── cleanup.sql                  # 清理腳本
└── application-test.yml             # 測試配置 (H2)
```

**SQL 腳本範例：**

```sql
-- employee_base_data.sql
INSERT INTO department (id, name, is_deleted) VALUES
    ('DEPT-001', '研發部', 0),
    ('DEPT-002', '人資部', 0);

INSERT INTO employee (id, name, department_id, status, is_deleted) VALUES
    ('EMP-001', '張三', 'DEPT-001', 'ACTIVE', 0),
    ('EMP-002', '李四', 'DEPT-001', 'ACTIVE', 0),
    ('EMP-003', '王五', 'DEPT-002', 'INACTIVE', 0),
    ('EMP-004', '趙六', 'DEPT-002', 'ACTIVE', 1);  -- 軟刪除

-- 測試場景：
-- 1. 查詢 ACTIVE 員工 → 預期 2 筆 (EMP-001, EMP-002)
-- 2. 查詢 DEPT-001 員工 → 預期 2 筆
-- 3. 查詢已刪除員工 → 預期 0 筆 (軟刪除過濾)
```
