# 快照測試方法論 (Snapshot Testing Methodology)

**版本:** 1.0
**建立日期:** 2025-12-30

---

## 概述

快照測試將預期結果存為 JSON 檔案，透過比對來驗證系統行為是否符合預期。

---

## 1. 核心流程

```
開發階段 (Learn Mode)
─────────────────────
1. 開發 Service 邏輯
2. 執行測試 (加上 -DupdateSnapshots=true)
3. 系統自動將結果存為 JSON 快照
4. Review 快照內容，確認正確
5. 提交快照到 Git

驗證階段 (Strict Mode)
─────────────────────
1. CI/CD 執行測試
2. 比對產出與快照
3. 不一致則報錯，顯示 Diff
4. 確認為預期變更 → 更新快照
5. 確認為 Bug → 修正程式碼
```

---

## 2. 快照範例

### QueryGroup 快照

```json
// employee_search_by_dept.json
{
  "junction": "AND",
  "conditions": [
    { "f": "department.name", "op": "EQ", "v": "研發部" },
    { "f": "status", "op": "EQ", "v": "ACTIVE" }
  ],
  "subGroups": []
}
```

### 計算結果快照

```json
// bonus_standard_case.json
{
  "finalValue": 50000,
  "appliedRules": [
    "BASE_SALARY_RATE: 月薪 x 1.5",
    "PERFORMANCE_BONUS: 績效加成 10%"
  ],
  "breakdown": {
    "baseSalary": 40000,
    "performanceBonus": 10000
  }
}
```

## 3. 處理動態值

某些欄位（如 ID、時間戳）每次執行都不同，需要忽略：

```java
SnapshotConfig config = SnapshotConfig.builder()
    .ignoreFields("id", "createdAt", "updatedAt")
    .ignorePattern(".*Id$")  // 忽略所有以 Id 結尾的欄位
    .build();

FluentAssert.assertMatchesSnapshot("result.json", actual, config);
```


---

## 5. 執行命令

```bash
# 驗證模式 (CI/CD)
mvn test

# 學習模式 (更新快照)
mvn test -DupdateSnapshots=true
```

## 6. 使用範例

```java
class UserServiceTest extends BaseServiceTest {

    @Test
    void getActiveUsers_ShouldMatchSnapshot() {
        GetUserListRequest req = new GetUserListRequest();
        req.setStatus("ACTIVE");

        service.getResponse(req, mockUser);

        // 一行驗證
        FluentAssert.assertMatchesSnapshot(
            "query/iam/user_active_search.json",
            service.getLastQueryGroup()
        );
    }
}
```
