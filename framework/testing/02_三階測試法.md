# 三階測試法 (Three-Tier Testing Method)

**版本:** 1.0
**建立日期:** 2025-12-30

---

## 概述

三階測試法是本架構的核心測試策略，旨在以最低成本達成高覆蓋率驗證。

```
┌─────────────────────────────────────────────────────────────────┐
│                    三階測試法                                    │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  第一階：引擎契約測試 (Engine Contract Test)                     │
│  ─────────────────────────────────────────                      │
│  對象：QueryEngine                                              │
│  做法：H2 固定資料集，測試所有 Operator                          │
│  目的：確保「翻譯機」不會壞，後續不需跑 DB                        │
│                                                                 │
│  第二階：業務組裝測試 (Business Assembly Test)                   │
│  ─────────────────────────────────────────                      │
│  對象：Application Service                                      │
│  做法：攔截 QueryGroup，驗證條件組裝是否正確                     │
│  工具：快照比對 或 DSL 斷言                                      │
│                                                                 │
│  第三階：合約驅動測試 (Contract-Driven Test)                     │
│  ─────────────────────────────────────────                      │
│  對象：API → QueryGroup 完整流程                                │
│  做法：解析 Markdown 合約，驗證是否符合 SA 規格                  │
│  目的：確保程式完全符合業務需求                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 1. 第一階：引擎契約測試

**目的**：一次性驗證 QueryEngine 的所有操作符，後續業務測試不再碰 DB。

```java
@SpringBootTest
class QueryEngineContractTest {

    @ParameterizedTest
    @MethodSource("operatorTestCases")
    void operator_ShouldProduceCorrectResult(Operator op, Object value, int expectedCount) {
        QueryGroup query = QueryBuilder.where()
            .and("status", op, value)
            .build();

        List<Entity> result = repository.findAll(query);

        assertThat(result).hasSize(expectedCount);
    }

    static Stream<Arguments> operatorTestCases() {
        return Stream.of(
            Arguments.of(Operator.EQ, "ACTIVE", 10),
            Arguments.of(Operator.NE, "ACTIVE", 5),
            Arguments.of(Operator.LIKE, "John%", 3),
            Arguments.of(Operator.IN, List.of("A", "B"), 7),
            Arguments.of(Operator.IS_NULL, null, 2),
            Arguments.of(Operator.GTE, 100, 8)
        );
    }
}
```

## 2. 第二階：業務組裝測試

**目的**：驗證 Service 組裝的 QueryGroup 是否符合業務意圖，使用快照比對。

```java
class EmployeeQueryServiceTest extends BaseServiceTest {

    @Test
    void searchEmployee_ShouldBuildCorrectQuery() {
        GetEmployeeListRequest req = new GetEmployeeListRequest();
        req.setDepartmentName("研發部");
        req.setStatus("ACTIVE");

        service.getResponse(req, mockUser);

        // 快照比對
        FluentAssert.assertMatchesSnapshot(
            "employee_search_by_dept.json",
            service.getLastQueryGroup()
        );
    }
}
```

## 3. 第三階：合約驅動測試

**目的**：驗證 API 到 QueryGroup 的完整流程符合 SA 定義的業務合約。

合約規格文件位於：`knowledge/05_Testing_Standards/Contracts/`

```java
class EmployeeContractTest extends BaseContractTest {

    @Test
    void searchEmployee_ShouldMatchContract() throws Exception {
        String contract = loadContractSpec("employee");

        EmployeeSearchRequest req = new EmployeeSearchRequest();
        req.setStatus("ACTIVE");

        verifyApiContract("/api/v1/employees/search", req, contract, "EMP-001");
    }
}
```
