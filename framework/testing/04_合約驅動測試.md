# 合約驅動測試 (Contract-Driven Testing)

**版本:** 1.0
**建立日期:** 2025-12-30

---

## 概述

合約驅動測試將「需求」直接轉化為「測試斷言」，減少人為轉譯損失。SA 在 Markdown 定義業務場景，工程師的程式碼必須符合這些合約。

---

## 1. 運作流程

1. **規格定義 (Define):** SA 在 Markdown 定義業務場景與必須包含的過濾條件
2. **實作 (Implement):** 工程師根據需求開發 Controller 與 Service
3. **攔截 (Intercept):** 測試執行時，攔截 Service 產出的 QueryGroup
4. **解析 (Parse):** ContractEngine 解析 Markdown 表格內容
5. **斷言 (Assert):** 比對 QueryGroup 是否滿足 Markdown 的合約要求

---

## 2. Markdown 合約格式

合約規格文件位於：`knowledge/05_Testing_Standards/Contracts/`

範例 (`iam_contracts.md`)：

```markdown
### 使用者查詢合約 (User Query Contract)

| 場景 ID | 測試描述 | 模擬角色 | 輸入 (Request) | 必須包含的過濾條件 |
| :--- | :--- | :--- | :--- | :--- |
| IAM_U001 | 查詢啟用中的使用者 | ADMIN | `{"status":"ACTIVE"}` | `status = 'ACTIVE'`, `is_deleted = 0` |
| IAM_U002 | 依帳號模糊查詢 | ADMIN | `{"username":"admin"}` | `username LIKE 'admin'`, `is_deleted = 0` |
```

---

## 3. 測試範例

```java
class UserContractTest extends BaseContractTest {

    @Test
    void searchActiveUsers_ShouldMatchContract() throws Exception {
        // 1. 載入合約規格
        String contract = loadContractSpec("iam");

        // 2. 準備請求
        GetUserListRequest req = new GetUserListRequest();
        req.setStatus("ACTIVE");

        // 3. 驗證合約 (場景 IAM_U001)
        verifyApiContract("/api/v1/users", req, contract, "IAM_U001");
    }
}
```

---

## 4. 通用安全規則

根據合約規格，所有查詢必須遵守：

1. **所有查詢都必須包含 `is_deleted = 0`** (登入紀錄除外)
2. **租戶隔離**: 非 SUPER_ADMIN 只能查詢自己租戶的資料
3. **帳號狀態過濾**: 依據查詢目的正確過濾狀態

---

## 5. 核心價值

1. **非對稱驗證:** 工程師不再是自己測試自己，而是由 SA 定義的 Markdown 作為「外部法規」
2. **業務可視化:** 所有過濾邏輯都顯性化地寫在 Markdown 中
3. **溝通革命:** PM/SA 看到測試失敗時，會知道是「合約未達成」
4. **零 Mock 負擔:** 透過 QueryGroup 攔截技術，測試不需啟動資料庫
