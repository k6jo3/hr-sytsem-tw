# 命名規範

**版本:** 1.1
**目的:** 統一專案命名標準，實現「程式即文檔」

---

## 1. 命名哲學

> 好的命名讓程式碼具備自解釋性，新人無需額外說明即可理解

### 1.1 命名原則

| 原則 | 說明 | 範例 |
|:---|:---|:---|
| **明確意圖** | 名稱應表達「做什麼」而非「怎麼做」 | `calculateBonus` > `processData` |
| **一致性** | 相同概念使用相同詞彙 | 統一用 `create` 或 `add` |
| **可讀性** | 寧可長一點也要清楚 | `employeeTerminationDate` > `empTermDt` |
| **領域語言** | 使用業務術語 | `Invoice` > `Bill` |

## 3. 後端命名規範

### 3.1 Controller

**命名格式:** `HR{Domain代號}{畫面名稱}{Controller類型}Controller`

| 類型 | 代碼 | 範例 | 說明 |
|:---|:---|:---|:---|
| Command | `Cmd` | `HR01UserCmdController` | 處理寫入操作 |
| Query | `Qry` | `HR01UserQryController` | 處理查詢操作 |

**繼承關係:**
- Command Controller → `CommandBaseController`
- Query Controller → `QueryBaseController`

**範例對照:**

| Domain | 畫面 | Controller名稱 | 說明 |
|:---|:---|:---|:---|
| IAM (01) | User | `HR01UserCmdController` | 使用者管理寫入 |
| ORG (02) | Employee | `HR02EmployeeQryController` | 員工管理查詢 |
| ATT (03) | Leave | `HR03LeaveCmdController` | 請假管理寫入 |

### 3.2 Service

| 類型 | 格式 | 範例 | Bean Name |
|:---|:---|:---|:---|
| Command | `{動詞}{名詞}ServiceImpl` | `CreateUserServiceImpl` | `createUserServiceImpl` |
| Query | `Get{名詞}{Type}ServiceImpl` | `GetUserListServiceImpl` | `getUserListServiceImpl` |

**常用動詞對照:**

| 動詞 | 用途 | 範例 |
|:---|:---|:---|
| `Create` | 新增 | `CreateUserServiceImpl` |
| `Update` | 修改 | `UpdateUserServiceImpl` |
| `Delete` | 刪除 | `DeleteUserServiceImpl` |
| `Get` | 查詢 | `GetUserDetailServiceImpl` |
| `Calculate` | 計算 | `CalculateBonusServiceImpl` |
| `Submit` | 提交 | `SubmitLeaveRequestServiceImpl` |
| `Approve` | 核准 | `ApproveLeaveRequestServiceImpl` |
| `Cancel` | 取消 | `CancelOrderServiceImpl` |

### 3.3 Domain 元件

| 元件 | 格式 | 範例 |
|:---|:---|:---|
| Aggregate Root | `{名詞}` | `User`, `Order`, `Employee` |
| Entity | `{名詞}` | `OrderItem`, `Address` |
| Value Object | `{名詞}` | `Email`, `Money`, `DateRange` |
| Domain Service | `{業務事件}DomainService` | `AccountLockingDomainService` |
| Domain Event | `{Aggregate}{PastVerb}Event` | `UserCreatedEvent`, `OrderConfirmedEvent` |
| Repository Interface | `I{名詞}Repository` | `IUserRepository` |

### 3.4 Infrastructure 元件

| 元件 | 格式 | 範例 |
|:---|:---|:---|
| Repository Impl | `{名詞}RepositoryImpl` | `UserRepositoryImpl` |
| DAO | `{名詞}DAO` | `UserDAO` |
| PO (Persistence Object) | `{名詞}PO` | `UserPO` |
| Mapper (MyBatis) | `{名詞}Mapper` | `UserMapper` |
| Assembler | `{名詞}Assembler` | `UserAssembler` |

### 3.5 DTO

| 類型 | 格式 | 範例 |
|:---|:---|:---|
| Request | `{動詞}{名詞}Request` | `CreateUserRequest`, `GetUserRequest` |
| Response (詳情) | `{名詞}DetailResponse` | `UserDetailResponse` |
| Response (列表) | `{名詞}ListResponse` | `UserListResponse` |
| Response (摘要) | `{名詞}SummaryResponse` | `OrderSummaryResponse` |

### 3.6 Pipeline 元件

| 元件 | 格式 | 範例 |
|:---|:---|:---|
| Context | `{UseCase}Context` | `SalaryContext`, `CreateUserContext` |
| Infrastructure Task | `Load{Entity}Task` | `LoadEmployeeTask`, `LoadAttendanceTask` |
| Domain Task | `{動詞}{業務}Task` | `CalculateTaxTask`, `ValidateEmailTask` |
| Integration Task | `{動詞}{Service}Task` | `SendNotificationTask` |

### 3.7 Exception

| 格式 | 範例 | 說明 |
|:---|:---|:---|
| `{描述}Exception` | `AccountLockedException` | 業務例外 |
| `{Entity}NotFoundException` | `UserNotFoundException` | 資源不存在 (404) |
| `{Action}FailedException` | `PaymentFailedException` | 操作失敗 |
| `Duplicate{Entity}Exception` | `DuplicateUserException` | 資源衝突 (409) |
| `ValidationException` | `ValidationException` | 參數驗證失敗 (400) |

## 4. 前端命名規範

### 4.1 檔案命名

| 類型 | 格式 | 範例 |
|:---|:---|:---|
| Page Component | `{Domain}{PageName}Page.tsx` | `HR01UserListPage.tsx` |
| Feature Component | `{Name}.tsx` (PascalCase) | `EmployeeCard.tsx` |
| Hook | `use{Name}.ts` | `useUserList.ts` |
| Factory | `{Name}Factory.ts` | `UserViewModelFactory.ts` |
| API Module | `{feature}Api.ts` | `userApi.ts` |
| Types | `{feature}Types.ts` | `userTypes.ts` |

### 4.2 元件命名

| 類型 | 格式 | 範例 |
|:---|:---|:---|
| Page | `{Domain}{Page}Page` | `HR01UserListPage` |
| Container | `{Feature}Container` | `UserListContainer` |
| Presentational | `{Name}` | `UserCard`, `UserTable` |
| Modal | `{Action}{Entity}Modal` | `CreateUserModal` |
| Form | `{Entity}Form` | `UserForm` |

### 4.3 Hook 命名

| 用途 | 格式 | 範例 |
|:---|:---|:---|
| 資料查詢 | `use{Entity}List` | `useUserList` |
| 單筆查詢 | `use{Entity}Detail` | `useUserDetail` |
| 表單狀態 | `use{Entity}Form` | `useUserForm` |
| 通用操作 | `use{Action}` | `useLogin`, `useLogout` |

### 4.4 變數/函數命名

| 類型 | 格式 | 範例 |
|:---|:---|:---|
| Boolean | `is{State}`, `has{Feature}` | `isLoading`, `hasPermission` |
| Handler | `handle{Event}` | `handleSubmit`, `handleClick` |
| Callback | `on{Event}` | `onSuccess`, `onChange` |
| Getter | `get{Value}` | `getFullName` |

## 5. API 路徑規範

### 5.1 基本格式

```
/api/v{version}/{resource-plural}/{id}/{sub-resource}/{action}
```

### 5.2 範例

| 操作 | Method | Path |
|:---|:---|:---|
| 列表查詢 | GET | `/api/v1/employees` |
| 單筆查詢 | GET | `/api/v1/employees/{id}` |
| 新增 | POST | `/api/v1/employees` |
| 修改 | PUT | `/api/v1/employees/{id}` |
| 刪除 | DELETE | `/api/v1/employees/{id}` |
| 子資源 | GET | `/api/v1/employees/{id}/leaves` |
| 動作 | POST | `/api/v1/employees/{id}/terminate` |

### 5.3 規則

- 資源名稱使用**複數名詞**
- 使用 **kebab-case** (小寫 + 連字號)
- 動作放在資源後面

## 6. 資料庫命名規範

### 6.1 表格

| 格式 | 範例 |
|:---|:---|
| `{module}_{entity}` (複數) | `iam_users`, `org_employees` |

### 6.2 欄位

| 類型 | 格式 | 範例 |
|:---|:---|:---|
| 主鍵 | `{table_singular}_id` | `user_id` (推薦) 或 `id` |
| 外鍵 | `{entity}_id` | `department_id` |
| 時間戳 | `{action}_at` | `created_at`, `updated_at` |
| 狀態 | `{noun}_status` | `user_status` |
| 布林 | `is_{state}` | `is_deleted`, `is_active` |

## 7. 命名檢查清單

### Code Review 時檢查

- [ ] Controller 是否符合 `HR{DD}{Screen}{Type}Controller` 格式？
- [ ] Service 是否符合 `Command/Query` 分離命名？
- [ ] 是否使用業務術語 (Domain Language)？
- [ ] 是否有縮寫需要展開？
- [ ] Boolean 變數是否以 `is/has` 開頭？
