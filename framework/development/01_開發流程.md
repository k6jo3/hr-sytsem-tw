## 第一階段：定義契約與測試 (Contract & Test First)

> 這是確保「程式照需求跑」的關鍵步驟

### 1.1 解析需求規格

SA 定義好業務場景（如：員工查詢、加班費計算規則）

**輸入:** Markdown 規格書、合約表格
**輸出:** 預期的 QueryGroup 或業務結果快照

### 1.2 撰寫失敗的測試 (RED)

根據規範建立測試案例，先定義「正確的結果」長什麼樣子。

```java
@Test
void searchEmployee_ByDepartment_ShouldMatchContract() {
    // 1. 載入 SA 定義的合約
    String contract = loadContractSpec("employee");

    // 2. 模擬請求
    EmployeeSearchRequest req = new EmployeeSearchRequest();
    req.setDepartmentId("D001");

    // 3. 執行並驗證合約
    // 此時測試會失敗 (RED) - 因為尚未實作
    verifyApiContract("/api/v1/employees/search", req, contract, "EMP-001");
}
```

**目的:** 先決定終點，再開始開發

## 第二階段：領域建模 (Domain Modeling)

> 在動手寫 API 之前，先建立核心業務邏輯的基礎

### 2.1 建立 Domain Entity 與 Aggregate Root

根據 DDD 規範，定義純淨的 Java 物件。

```java
// Aggregate Root
public class Employee extends AggregateRoot<EmployeeId> {

    private String firstName;
    private String lastName;
    private Email email;
    private DepartmentId departmentId;
    private EmployeeStatus status;

    // Factory Method
    public static Employee create(CreateEmployeeData data) {
        Employee emp = new Employee();
        emp.firstName = data.getFirstName();
        emp.lastName = data.getLastName();
        emp.email = new Email(data.getEmail());
        emp.departmentId = data.getDepartmentId();
        emp.status = EmployeeStatus.ACTIVE;

        emp.registerEvent(new EmployeeCreatedEvent(emp));
        return emp;
    }

    // 業務方法
    public void terminate(String reason) {
        if (this.status != EmployeeStatus.ACTIVE) {
            throw new IllegalStateException("只有在職員工可離職");
        }
        this.status = EmployeeStatus.TERMINATED;
        this.registerEvent(new EmployeeTerminatedEvent(this, reason));
    }
}
```

### 2.2 建立 Domain Service

若邏輯涉及多個領域物件，則建立 Domain Service。

```java
// 命名格式: {業務事件}DomainService
public class SalaryCalculationDomainService {

    public CalculationResult<BigDecimal> calculateMonthly(
            Employee employee,
            AttendanceRecord attendance,
            OvertimeRecord overtime,
            SalaryPolicy policy) {

        // 複雜的跨領域計算邏輯
        return CalculationResult.<BigDecimal>builder()
            .finalValue(totalSalary)
            .addRule("BASE_SALARY", "基本薪資")
            .addRule("OVERTIME_PAY", "加班費")
            .breakdown("base", baseSalary)
            .breakdown("overtime", overtimePay)
            .build();
    }
}
```

**核心原則:** 這裡只處理「純業務規則」，不涉及資料庫存取

## 第三階段：流水線拆解 (Context & Tasks)

> 將複雜的業務行為分解為可管理的零件

### 3.1 定義 Context

建立該 API 專屬的 Context 物件，用來傳遞數據。

```java
@Data
public class CalculateSalaryContext {
    // 輸入
    private String employeeId;
    private YearMonth targetMonth;

    // 中間數據
    private Employee employee;
    private AttendanceRecord attendance;
    private OvertimeRecord overtime;
    private SalaryPolicy policy;

    // 輸出
    private CalculationResult<BigDecimal> result;
}
```

### 3.2 實作 Tasks

將業務邏輯拆解為細小的 Task：

**Infrastructure Task** - 負責資料存取：

```java
public class LoadEmployeeTask implements Task<CalculateSalaryContext> {

    private final EmployeeRepository repository;

    @Override
    public void execute(CalculateSalaryContext ctx) {
        Employee emp = repository.findById(ctx.getEmployeeId())
            .orElseThrow(() -> new NotFoundException("員工不存在"));
        ctx.setEmployee(emp);
    }
}
```

**Domain Task** - 負責調用 Domain Service 進行計算：

```java
public class CalculateSalaryTask implements Task<CalculateSalaryContext> {

    private final SalaryCalculationDomainService domainService;

    @Override
    public void execute(CalculateSalaryContext ctx) {
        CalculationResult<BigDecimal> result = domainService.calculateMonthly(
            ctx.getEmployee(),
            ctx.getAttendance(),
            ctx.getOvertime(),
            ctx.getPolicy()
        );
        ctx.setResult(result);
    }
}
```

## 第四階段：API 組裝與實作 (Application Integration)

> 最後一步才是將零件組合成服務

### 4.1 建立 Service Impl

建立 `{動詞}{名詞}ServiceImpl`，並明確指定 Bean 名稱。

```java
@Service("calculateMonthlySalaryServiceImpl")  // 必須明確指定
@Transactional
@RequiredArgsConstructor
public class CalculateMonthlySalaryServiceImpl
        extends BaseCommandService<CalculateSalaryRequest, SalaryCalculationResponse> {

    private final LoadEmployeeTask loadEmployeeTask;
    private final LoadAttendanceTask loadAttendanceTask;
    private final LoadOvertimeTask loadOvertimeTask;
    private final CalculateSalaryTask calculateSalaryTask;
    private final SalaryMapper mapper;
}
```

### 4.2 編排 Pipeline

在 `doExecute` 中，使用 Pipeline 照序連結所有 Task。

```java
@Override
protected SalaryCalculationResponse doExecute(
        CalculateSalaryRequest request,
        JWTModel currentUser,
        String... args) throws Exception {

    // 1. 建立 Context
    CalculateSalaryContext ctx = new CalculateSalaryContext();
    ctx.setEmployeeId(request.getEmployeeId());
    ctx.setTargetMonth(request.getTargetMonth());

    // 2. 執行 Pipeline
    BusinessPipeline.create(ctx)
        .addTask(loadEmployeeTask)
        .addTask(loadAttendanceTask)
        .addTask(loadOvertimeTask)
        .addTask(calculateSalaryTask)
        .execute();

    // 3. 轉換回應
    return mapper.toResponse(ctx.getResult());
}
```

### 4.3 實作 Controller

建立 Controller 並繼承 `BaseController`。

```java
@RestController
@RequestMapping("/api/v1/payroll")
public class PayrollCmdController extends CommandBaseController {

    @PostMapping("/calculate")
    @Operation(summary = "計算月薪", operationId = "calculateMonthlySalary")
    public ResponseEntity<SalaryCalculationResponse> calculateMonthlySalary(
            @RequestBody @Valid CalculateSalaryRequest request,
            @CurrentUser JWTModel currentUser) throws Exception {

        // Controller 只需調用 execCommand
        return ResponseEntity.ok(execCommand(request, currentUser));
    }
}
```

**註:** Controller 僅需調用 `execCommand(request, currentUser)` 即可完成實作

## 第五階段：驗證與重構 (Green & Refactor)

### 5.1 跑測試 (GREEN)

確保剛才寫的代碼能通過第一階段定義的合約測試。

```bash
# 執行測試
mvn test

# 若快照需更新
mvn test -DupdateSnapshots=true
```

### 5.2 重構 (REFACTOR)

優化代碼結構，確保符合以下標準：

**SOLID 原則檢查:**

- [ ] **S** - 每個 Task 只做一件事
- [ ] **O** - Pipeline 可擴展新 Task
- [ ] **L** - 子類別可替換父類別
- [ ] **I** - 介面最小化
- [ ] **D** - 依賴介面而非實作

**命名規範檢查:**

- [ ] Controller: `{Domain}{Entity}CmdController`
- [ ] Service: `{動詞}{名詞}ServiceImpl`
- [ ] Task: `{動作}{Entity}Task`
- [ ] Context: `{UseCase}Context`
