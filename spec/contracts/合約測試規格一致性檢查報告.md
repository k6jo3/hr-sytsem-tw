## 摘要

本報告檢查 HR01 (IAM) 與 HR02 (組織員工) 服務的三份文件一致性：
- **需求分析書 (SA/)**: 定義業務需求與 API 規格
- **系統設計書 (spec/)**: 定義 UI/UX 設計與技術實作細節
- **開發合約 (spec/contracts/)**: 定義查詢場景與測試條件

### 檢查結果總覽

|:---|:---:|:---:|:---:|:---:|
| HR01 IAM | 15 | 8 | 3 | 需同步修正 |
| HR02 組織員工 | 18 | 10 | 4 | 需同步修正 |

## 修正建議

### 🔴 嚴重問題 - 必須修正

#### HR01 IAM 服務

1. **新增 User 清單查詢 API 至需求分析書**
   ```
   GET /api/v1/users
   Query Parameters: search, status, roleId, tenantId, page, size
   ```

2. **移除合約的 department_id 過濾條件 (IAM_U006)**
   - User 表不應有 department_id
   - 若需部門過濾，應透過 Organization Service 關聯查詢

3. **統一 Role 的系統/自訂角色欄位**
   - 建議使用需求書的 `is_system_role` (Boolean)
   - 合約修改為 `is_system_role = true/false`

4. **統一 Login Log 的結果欄位**
   - 建議使用需求書的 `success` (Boolean)
   - 合約修改為 `success = true/false`

5. **新增 Role 和 Permission 的 is_deleted 軟刪除欄位**
   - 同步新增至需求書和設計書

#### HR02 組織員工服務

1. **新增 Employee 清單查詢 API 至需求分析書**
   ```
   GET /api/v1/employees
   Query Parameters: search, status, departmentId, hireDateFrom, hireDateTo, page, size
   ```

2. **新增 Position (職位) 領域模型**
   - 需求分析書新增 Position 實體
   - 系統設計書新增 positions 資料表
   - 定義 Position API

3. **統一 Employee 狀態值**
   | 建議值 | 說明 |
   |:---|:---|
   | PROBATION | 試用期 |
   | ACTIVE | 在職 |
   | ON_LEAVE | 留職停薪 |
   | TERMINATED | 離職 |

4. **新增匯入/匯出 API 至需求分析書**
   ```
   POST /api/v1/employees/import
   GET /api/v1/employees/export
   ```

5. **新增唯一性檢查 API 至需求分析書**
   ```
   GET /api/v1/employees/check-number
   GET /api/v1/employees/check-national-id
   GET /api/v1/employees/check-email
   ```

### ⚠️ 中度問題 - 建議修正

1. **統一欄位命名規範**
   - `employee_number` vs `employee_no` → 統一使用 `employee_number`
   - `department_id` vs `dept_id` → 統一使用 `department_id`
   - `full_name` vs `name` → 統一使用 `full_name`

2. **補充認證相關的測試合約**
   - 新增 login/logout/refresh/verify 的測試場景

3. **補充 Command API 的測試合約**
   - 建立/更新/刪除操作的測試場景

### ✅ 已一致項目

- 認證流程 (login/logout)
- 員工生命週期事件
- 部門階層結構
- 租戶隔離機制
- 權限控管架構
