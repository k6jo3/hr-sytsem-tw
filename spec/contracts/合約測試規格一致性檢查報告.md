## 摘要

本報告檢查 HR01~HR04 服務的三份文件一致性：
- **需求分析書 (SA/)**: 定義業務需求與 API 規格
- **系統設計書 (spec/)**: 定義 UI/UX 設計與技術實作細節

### 4. 設計書與需求書特殊說明

#### 4.1 獎金與扣款處理方式差異

**設計書實作方式:**
- 獎金與扣款內嵌於 `salary_structure_items` 表
- 薪資單內的獎金與扣款透過 JSONB 欄位 (`earnings`/`deductions`) 儲存
- 提供彈性但不利於獨立查詢

**合約定義方式:**
- 獎金 (Bonus) 作為獨立實體
- 扣款 (Deduction) 作為獨立實體
- 有明確的狀態管理 (ACTIVE/COMPLETED/PAID)

#### 4.2 日期格式差異

| 來源 | 格式 | 範例 |
|:---|:---|:---|
| 設計書 | DATE 類型 | `pay_period_start: 2025-01-01`, `pay_period_end: 2025-01-31` |
| 合約 | VARCHAR 年月 | `year_month: '2025-01'` |

#### 4.3 狀態值對照表

**payslips.status:**
| 設計書 | 合約 | 說明 |
|:---|:---|:---|
| DRAFT | - | 草稿 |
| FINALIZED | PENDING | 已定案待發放 |
| SENT | PAID | 已發送/已發放 |
