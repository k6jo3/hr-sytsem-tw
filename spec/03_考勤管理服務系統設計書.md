## 1. 服務概述

### 1.1 服務定位
考勤管理服務負責員工的出勤管理、假勤申請與審核，以及加班管理。本服務必須確保**符合台灣勞動基準法**的所有規定。

### 1.2 核心功能
- ✅ **打卡管理:** 多元打卡方式、打卡異常偵測、補卡申請
- ✅ **班別設定:** 彈性班別、輪班、彈性工時管理
- ✅ **假勤管理:** 20+種假別設定、請假申請與審核、假期餘額計算
- ✅ **特休假管理:** 依勞基法自動計算特休、到期提醒、未休補償
- ✅ **加班管理:** 加班申請審核、加班時數管控（46小時/月、138小時/3月）
- ✅ **變形工時:** 支援二週/四週/八週變形工時排班
- ✅ **法規遵循:** 女性員工保護、職業災害管理
- ✅ **差勤報表:** 個人/部門月度差勤統計

### 1.3 技術架構
- **前端:** ReactJS + Redux + Ant Design
- **後端:** Spring Boot 3.1.x + MyBatis
- **資料庫:** PostgreSQL 15.x
- **事件匯流排:** Kafka
- **排程任務:** Quartz Scheduler

### 1.4 服務邊界

| 屬於本服務 | 不屬於本服務 |
|:---|:---|
| 打卡記錄 | 薪資計算 (Payroll Service使用本服務數據) |
| 請假申請與審核 | 員工基本資料 (Organization Service) |
| 加班申請與審核 | 審核流程引擎 (Workflow Service) |
| 假期餘額計算 | |
| 班別與假別規則設定 | |

## 3. UX流程設計

### 3.1 員工打卡流程

```mermaid
sequenceDiagram
    actor Employee as 員工
    participant Page as 打卡頁面
    participant API as Attendance API
    participant DB as PostgreSQL
    participant Kafka as Kafka
    
    Employee->>Page: 1. 開啟打卡頁面
    Page->>Page: 2. 取得GPS定位
    Page->>Page: 3. 取得設備IP
    Page->>API: 4. 載入今日班別資訊
    API->>DB: 5. 查詢員工班別
    DB-->>API: 6. 返回班別
    API-->>Page: 7. 顯示班別與今日記錄
    
    Employee->>Page: 8. 點擊「上班打卡」
    Page->>API: 9. POST /api/v1/attendance/check-in
    API->>API: 10. 驗證GPS範圍
    API->>API: 11. 檢查是否遲到
    API->>DB: 12. INSERT attendance_records
    
    alt 打卡正常
        API-->>Page: 13a. 返回成功
        Page->>Employee: 14a. 顯示「打卡成功」
    else 打卡異常(遲到)
        API->>Kafka: 13b. Publish AttendanceAnomalyDetected
        API-->>Page: 14b. 返回成功 + 遲到警告
        Page->>Employee: 15b. 顯示「已打卡，遲到5分鐘」
    end
```

**關鍵點:**
- ✅ GPS定位驗證 (誤差<50m)
- ✅ 自動偵測遲到/早退
- ✅ 異常打卡發布事件通知

### 3.2 請假申請流程

```mermaid
sequenceDiagram
    actor Employee as 員工
    participant Page as 請假申請頁面
    participant Att as Attendance API
    participant Workflow as Workflow Service
    participant DB as PostgreSQL
    participant Kafka
    participant Manager as 主管
    participant Payroll as Payroll Service
    
    Employee->>Page: 1. 開啟請假申請頁面
    Page->>Att: 2. GET /api/v1/leave/balances
    Att-->>Page: 3. 返回假期餘額
    Page->>Employee: 4. 顯示可請假別與餘額
    
    Employee->>Page: 5. 填寫請假表單
    Employee->>Page: 6. 點擊「送出申請」
    Page->>Att: 7. POST /api/v1/leave/applications
    
    Att->>Att: 8. 驗證餘額足夠
    Att->>Att: 9. 驗證日期不重疊
    Att->>Att: 10. 計算請假天數
    Att->>DB: 11. INSERT leave_applications (PENDING)
    Att->>Workflow: 12. 建立審核流程
    Att->>Kafka: 13. Publish LeaveApplied
    Att-->>Page: 14. 返回待審核狀態
    Page->>Employee: 15. 顯示申請成功
    
    Note over Workflow,Manager: 審核流程
    
    Manager->>Att: 16. PUT /approve
    Att->>DB: 17. UPDATE status=APPROVED
    Att->>DB: 18. UPDATE leave_balance (扣除)
    Att->>Kafka: 19. Publish LeaveApproved
    
    Kafka->>Payroll: 20. 訂閱事件
    Payroll->>Payroll: 21. 記錄請假扣薪資料
```

**關鍵點:**
- ✅ 即時餘額驗證
- ✅ 重疊日期檢查
- ✅ 整合Workflow審核流程
- ✅ 核准後自動扣除餘額
- ✅ 發布事件供Payroll計算

### 3.3 加班申請流程

```mermaid
sequenceDiagram
    actor Employee as 員工
    participant Page as 加班申請頁面
    participant Att as Attendance API
    participant DB as PostgreSQL
    participant Kafka
    participant Manager as 主管
    participant Payroll as Payroll Service
    
    Employee->>Page: 1. 填寫加班申請
    Page->>Att: 2. POST /api/v1/overtime/applications
    
    Att->>DB: 3. 查詢本月累計加班時數
    Att->>Att: 4. 檢查是否超過46小時/月
    Att->>Att: 5. 檢查是否超過138小時/季
    
    alt 超過限制
        Att->>Kafka: 6a. Publish OvertimeLimitExceeded
        Att-->>Page: 7a. 返回警告 (可設定阻擋或警告)
    else 未超過
        Att->>DB: 6b. INSERT overtime_applications
        Att-->>Page: 7b. 返回待審核
    end
    
    Manager->>Att: 8. PUT /approve
    Att->>DB: 9. UPDATE status=APPROVED
    Att->>Kafka: 10. Publish OvertimeApproved
    
    Kafka->>Payroll: 11. 訂閱事件
    Payroll->>Payroll: 12. 計算加班費
```

**關鍵點:**
- ✅ 勞基法加班時數管控 (46h/月、138h/季)
- ✅ 超時申請預警機制
- ✅ 審核後自動計算加班費/補休
